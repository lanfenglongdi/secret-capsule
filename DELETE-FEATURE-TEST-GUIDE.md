# 删除秘密功能测试指南

## 📋 功能概述
本功能允许用户在验证密码后删除已创建的秘密。这是一个**可选功能**，仅用于本地测试，不会上传到任何网络。

## ⚠️ 重要提示
- **此功能仅在本地测试分支 `feature-delete-secret-local-test` 上可用**
- **不会合并到 main-chinese 或 main 分支**
- **不会部署到阿里云或 Vercel**
- **符合"秘密胶囊"的核心理念：默认情况下秘密不可删除**

##  使用方法

### 1. 启动本地服务器
```bash
cd d:\secret-capsule
npm run dev
```
服务器将运行在 `http://localhost:3001`

### 2. 创建一个测试秘密
1. 访问 `http://localhost:3001/create`
2. 输入秘密内容和密码
3. 完成人机验证
4. 点击"创建秘密"
5. 复制生成的秘密编号（例如：SC-ABC123）

### 3. 解锁并删除秘密
1. 访问 `http://localhost:3001/s/SC-ABC123`（替换为您的秘密编号）
2. 输入密码
3. 点击"立即解锁"
4. 查看解密后的秘密内容
5. 点击底部的"🗑️ 删除此秘密"按钮
6. 在确认对话框中再次输入密码
7. 点击"确认删除"

### 4. 验证删除结果
- 如果删除成功，会显示"✅ 秘密已成功删除！"提示
- 尝试再次访问该秘密编号，应该显示"秘密不存在"

## 🔒 安全机制

### 密码验证
- 删除秘密需要输入正确的密码
- 系统会尝试验证密码的正确性（通过解密测试）
- 如果密码错误，会显示"密码错误，无法删除"

### API 端点
- **删除 API**: `/api/delete`
- **请求方法**: POST
- **请求体**:
  ```json
  {
    "id": "SC-ABC123",
    "password": "your_password"
  }
  ```
- **响应**:
  - 成功: `{ "ok": true, "message": "秘密已删除" }`
  - 失败: `{ "error": "错误信息", "code": "错误代码" }`

### 错误代码
- `NOT_FOUND`: 秘密不存在
- `WRONG_PASSWORD`: 密码错误
- `VALIDATION_ERROR`: 输入验证失败
- `DELETE_ERROR`: 数据库删除失败
- `SERVER_ERROR`: 服务器内部错误

## 📝 注意事项

1. **数据持久化**: 删除操作会从 Supabase 数据库中永久移除数据
2. **不可恢复**: 删除后无法恢复秘密内容
3. **端到端加密**: 即使删除，服务器也无法看到秘密内容
4. **本地测试**: 此功能仅在本地测试，不会影响生产环境

## 🔄 清理测试数据

如果需要清理测试数据，可以：
1. 手动删除特定秘密（使用删除功能）
2. 清空 Supabase 数据库中的 secrets 表（仅限测试环境）
